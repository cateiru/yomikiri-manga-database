# 読み切り漫画データベース デザインドキュメント

[001_plan.md](./001_plan.md) で定義した Web サービスのビジュアルデザイン指針を定める。
UI 実装（`apps/web`）はこのドキュメントに従う。

## 1. デザインコンセプト

**「漫画のコマ」をメタファーにしたモノトーン UI。**

読み切り漫画を扱うサービスであることを、原稿用紙のような白地と
墨のような黒い枠線で表現する。カード・モーダル・ボタンといった UI 要素を
「コマ」に見立て、角張ったソリッドな輪郭とベタ塗りの影で構成する。

- 紙面（背景）は白、線とテキストは墨色の 2 色を基本とするモノトーン
- 主役はあくまで各作品のサムネイル。UI 自体は無彩色に徹し、作品画像を引き立てる
- 過剰な漫画モチーフ（集中線・スクリーントーン・吹き出し等）は使わず、
  「太い枠線」「角張った輪郭」「ベタ塗りの影」の 3 要素に絞る

## 2. デザイン原則

| 原則 | 内容 |
| --- | --- |
| 角張った輪郭 | `border-radius: 0` を基本とする。丸みは一切つけない（ファビコンのみ例外。3.1 章参照） |
| ソリッドな枠線 | UI 要素の輪郭は墨色の実線ボーダーで描く。淡い線やグラデーションは使わない |
| ベタ塗りの影 | `box-shadow` は blur を使わず、右下方向へのオフセットのみのベタ塗りとする |
| モノトーン基調 | 墨色 `#2e3142` と白 `#fff`、およびその中間グレーのみで構成する |
| 例外カラーの限定 | 有彩色の使用は「他社サービスのロゴ」「エラー表示」「お気に入りボタンの選択状態」のみに限定する |

ベタ塗りの影の基本形:

```css
/* blur・spread を持たないオフセットシャドウ */
box-shadow: 4px 4px 0 0 var(--color-ink);
```

## 3. デザイントークン

トークンは CSS カスタムプロパティとして `apps/web` のグローバル CSS に定義する。

### 3.1 カラー

| トークン | 値 | 用途 |
| --- | --- | --- |
| `--color-ink` | `#2e3142` | 枠線・本文テキスト・影・塗りつぶし背景 |
| `--color-bg` | `#fff` | ページ背景・コンポーネント背景 |
| `--color-ink-70` | `#6c6f7b` | サブテキスト（作者名・サービス名・補足） |
| `--color-ink-40` | `#abadb3` | disabled 状態・プレースホルダーテキスト |
| `--color-ink-15` | `#e0e0e3` | 区切り線・淡い枠線 |
| `--color-ink-5` | `#f5f5f6` | わずかな面の塗り分け（コードブロック背景等） |
| `--color-error` | `#c73e3e` | エラーテキスト・エラー枠線（例外カラー） |
| `--color-favorite` | `#e0a300` | お気に入りボタンの選択状態（例外カラー） |

- グレー段階は `--color-ink` と白の混色（`color-mix(in srgb, var(--color-ink) 70%, white)` 相当）。
  上表の hex はその計算結果を固定値化したもの
- 有彩色トークンは `--color-error` と `--color-favorite` の 2 つに限定する
- 他社サービスのロゴ・ファビコンは各社のブランドカラーをそのまま表示する。
  モノトーン化（グレースケールフィルタ等）による改変は行わない
- ファビコンのみ、角丸なし原則の例外として `border-radius: 4px` を許容する
  （元画像が角丸で作られている場合になじませるため）

### 3.2 タイポグラフィ

Web フォントは読み込まず、システムフォントのみを使用する。

```css
--font-sans:
  system-ui,
  -apple-system,
  "Hiragino Sans",
  "Yu Gothic UI",
  "Yu Gothic",
  "Noto Sans JP",
  sans-serif;
```

| トークン | サイズ | ウェイト | 用途 |
| --- | --- | --- | --- |
| `--text-xl` | 24px | 700 | ページタイトル（サービスロゴ） |
| `--text-lg` | 18px | 700 | モーダル見出し・セクション見出し |
| `--text-md` | 15px | 700 | カードの作品タイトル |
| `--text-sm` | 13px | 400 | 本文・作者名・サービス名 |
| `--text-xs` | 11px | 700 | ジャンルバッジ・チップ |

- 見出し・タイトル・バッジは太字（700）で「描き文字」のような強さを出す
- `line-height` は本文 1.7、見出し 1.4 を基本とする

### 3.3 ボーダー・シャドウ・角丸

| トークン | 値 | 用途 |
| --- | --- | --- |
| `--border-width` | `2px` | 標準の枠線（カード・ボタン・チップ） |
| `--border-width-bold` | `3px` | 強調枠線（モーダル・ヘッダー下線） |
| `--radius` | `0` | 全要素共通。角丸は使わない（ファビコンのみ例外。3.1 章参照） |
| `--shadow-sm` | `4px 4px 0 0 var(--color-ink)` | カード・ボタンの標準の影 |
| `--shadow-lg` | `8px 8px 0 0 var(--color-ink)` | モーダル・hover 時の浮き上がり |

### 3.4 スペーシング

4px を基数としたスケールを使用する。

| トークン | 値 |
| --- | --- |
| `--space-1` | 4px |
| `--space-2` | 8px |
| `--space-3` | 12px |
| `--space-4` | 16px |
| `--space-6` | 24px |
| `--space-8` | 32px |

## 4. コンポーネント仕様

[001_plan.md の 6 章「画面仕様」](./001_plan.md) に登場する UI 要素を対象とする。

### 4.1 読み切りカード

トップページの grid に並ぶ、1 作品 = 1 コマのカード。

- 白背景、`--border-width` の墨色枠線、`--shadow-sm` の影
- 構成（上から順に）
  1. サムネイル画像: 幅いっぱい、アスペクト比固定（1:1）、`object-fit: contain`。
     カード枠と画像の間に隙間は作らず、画像下辺に墨色の区切り線を引く
  2. 作品タイトル（`--text-md`、最大 2 行で省略 `line-clamp: 2`）+ お気に入りボタン
     （同じ行。ボタンは星アイコンのトグルで、選択時は `--color-favorite` で塗りつぶす）
  3. 作者名: `--text-sm`、`--color-ink-70`
  4. 掲載サービス名: `--text-sm`、`--color-ink-70`。サービスのファビコン（有彩色可）を先頭に添える
  5. ジャンルバッジ: 得票数上位 3 件を横並び。投票が無い場合は領域ごと非表示
- カード全体が外部ビューワーへのリンク（新規タブ）。
  ホバー時は影を `--shadow-lg` に拡大しつつ `translate(-2px, -2px)` して
  「コマが浮き上がる」動きをつける（`transition: 0.15s ease`）
- active（押下）時は逆に `translate(2px, 2px)` + 影を消し、紙面に沈む動きとする
- 既読（読了検知済み）の作品は専用のスタイルで区別する

### 4.2 ジャンルバッジ

カード上でジャンルを示す小さなラベル。2 パターンを使い分ける。

| パターン | 見た目 | 用途 |
| --- | --- | --- |
| 標準 | 白地 + 墨色枠線 + 墨色テキスト | 得票 2〜3 位のジャンル |
| 反転 | 墨色ベタ塗り + 白テキスト | 得票 1 位のジャンル |

- `--text-xs`、`padding: 2px 8px`、角丸なし
- バッジ自体はクリック不可の表示専用要素とする

### 4.3 ボタン

| 種別 | 通常 | hover | active | disabled |
| --- | --- | --- | --- | --- |
| primary | 墨色ベタ塗り + 白文字 + 墨色枠線 + `--shadow-sm` | 浮き上がり（カードと同じ動き） | 沈み込み（影なし） | `--color-ink-40` 塗り + 影なし |
| secondary | 白地 + 墨色枠線 + 墨色文字 + `--shadow-sm` | 同上 | 同上 | 文字・枠線を `--color-ink-40` に |

- 用途: primary は投票モーダルの「投票する」、secondary は「スキップ」
- 高さ 40px 目安、`--text-sm` の太字（700）

### 4.4 ジャンル選択チップ

投票モーダル内で使う複数選択可のトグル。

| 状態 | 見た目 |
| --- | --- |
| 未選択 | 白地 + 墨色枠線 + 墨色テキスト |
| 選択済み | 墨色ベタ塗り + 白テキスト |
| hover（未選択時） | 背景を `--color-ink-5` に |

- 実装はチェックボックス（`input[type="checkbox"]` + ラベル）とし、
  選択状態が色だけに依存しないよう選択済みチップの先頭に「✓」を表示する
- バッジより一回り大きい `padding: 6px 12px`、`--text-sm`
- トップページのジャンル絞り込みフィルタにも同じチップを再利用する
  （grid 上部に横並びで配置し、選択状態の見た目は投票モーダルと共通）

### 4.5 ジャンル投票モーダル

読了判定後に表示するモーダル。ひときわ大きな「コマ」として描く。

- 白背景、`--border-width-bold` の墨色枠線、`--shadow-lg` の影、角丸なし
- オーバーレイは `rgba(46, 49, 66, 0.6)`（`--color-ink` ベースの半透明）
- 構成
  1. 対象作品のサムネイル（小）+ タイトル
  2. 質問文「この読み切りはどんなジャンルでしたか？」（`--text-lg`）
  3. ジャンルチップ一覧（4.4、折り返しの横並び）
  4. フッター: 「スキップ」（secondary）と「投票する」（primary）を右寄せ
- 「投票する」はジャンル未選択の間 disabled とする

### 4.6 ヘッダー / フッター

- ヘッダー: 白背景。下辺に `--border-width-bold` の墨色ボーダーを引き、
  紙面の枠線のように見せる。左にサービス名（`--text-xl` 太字テキストロゴ）
- フッター: 上辺に `--border-width` のボーダー。
  コピーライト・リポジトリへのリンク等を `--color-ink-70` で表示

### 4.7 エラー・空状態表示

**エラー表示（例外カラー使用箇所）**

- 投票 API 失敗時など: 白地 + `--color-error` の枠線 + `--color-error` のテキストのボックス
- 有彩色を使うのはこのエラー表示と他社ロゴのみ。それ以外での赤の流用は禁止

**空状態表示（モノトーン）**

- フィルタ結果 0 件など: 破線ボーダー（`2px dashed var(--color-ink-40)`）の
  「空のコマ」を中央に置き、`--color-ink-70` のテキストで案内を表示

### 4.8 サムネイルプレースホルダー

サムネイル URL が無い、またはホットリンク不可で画像を表示できない場合
（[001_plan.md 11 章](./001_plan.md) 参照）の代替表示。

- カードのサムネイル領域と同サイズ、背景 `--color-ink-5`
- 中央に `--color-ink-40` で「NO IMAGE」テキストを表示（画像アイコン等は使わない）

## 5. アクセシビリティ・留意点

- コントラスト比
  - `--color-ink` on `--color-bg`: 約 12.8:1（WCAG AAA 相当）
  - `--color-ink-70` on `--color-bg`: 約 5.0:1（AA を満たす）。本文・サブテキストに使用可
  - `--color-ink-40` はコントラスト不足のため、disabled 表示と
    プレースホルダー等の補助表現に限定し、読ませるテキストには使わない
  - `--color-error` on `--color-bg`: 約 5.0:1（AA を満たす）
- フォーカスリング: `outline: 2px solid var(--color-ink); outline-offset: 2px;` で統一し、
  角張った矩形の outline で世界観を維持する。`outline: none` による無効化は禁止
- hover / active の浮き沈みアニメーションは `prefers-reduced-motion: reduce` 時に無効化する
- 選択・状態の表現は色のみに依存させない（チップの「✓」表示など）
