## 実装タスク

### 1. Cloudflare Workers（apps/web）

- `wrangler.jsonc` を本番用に整備する（プラン 004 で作成済みの設定を拡張）
- `DATABASE_URL` を `wrangler secret put` で登録する
- `opennextjs-cloudflare build && wrangler deploy` によるデプロイスクリプトを
  `package.json` に定義する
- ISR（`revalidate: 300`）が Workers 上で機能することを確認する
- カスタムドメインの割り当て（用意できる場合）

### 2. さくらVPS（apps/batch、Docker）

- Docker / Docker Compose をセットアップする（Node.js を VPS に直接インストールする必要はない）
- バッチは `apps/batch/Dockerfile` でイメージ化する。本番用の `compose.prod.yaml`
  （ローカル開発用の `compose.yaml` とは別ファイル。`db` / `wsproxy` は含まず `batch` のみ定義）を
  `docker compose -f compose.prod.yaml run --rm batch` として都度起動・都度破棄する形で実行する
  - `restart: always` による常駐化は採用しない。バッチは 1 回実行して終了する oneshot 処理のため、
    常駐させると正常終了のたびに即座に再起動され「毎日 00:05・12:05・18:05（JST）」の実行スケジュールと両立しない
- デプロイ方法: VPS 上で `git pull` + `docker compose -f compose.prod.yaml build batch` を行う
  簡易スクリプト（`deploy/bin/deploy.sh`）を用意する
- `DATABASE_URL` は環境ファイル（`/etc/yomikiri/batch.env`、`EnvironmentFile=`、パーミッション 600）で管理し、
  systemd 経由で `docker compose` の環境変数（`${DATABASE_URL}`）として引き渡す
- systemd unit を作成する（`deploy/systemd/`）
  - `yomikiri-batch.service`: `Type=oneshot` で `deploy/bin/run-batch.sh` を実行する。
    このスクリプトが `docker compose run --rm batch` を実行し、成功時のみ
    `/var/lib/yomikiri/last-success` を touch する
  - `yomikiri-batch.timer`: `OnCalendar` で毎日 00:05・12:05・18:05（JST）に起動、`Persistent=true`
- 監視は Mackerel（`mackerel-agent`）で行う（`deploy/mackerel/yomikiri-batch-checks.conf`）
  - `check-file-age` プラグインで `/var/lib/yomikiri/last-success` の鮮度を監視し、
    13 時間（最大実行間隔 12 時間 + マージン）以上更新がなければ警告、16 時間以上でクリティカルとする
    （実行漏れ・ハングの検知）
  - `systemctl is-failed yomikiri-batch.service` をラップしたカスタムチェックで、
    直近の実行失敗（終了コード非 0）を即時検知する
- ログは journald で確認する（`journalctl -u yomikiri-batch`）。バッチコンテナは `--rm` で
  実行のたびに破棄されるため、`docker compose logs` は実行中の一時的な確認にしか使えない
  （`docker compose run` をフォアグラウンドで実行することで、標準出力/標準エラーは
  journald 側にそのまま残る）

#### VPS セットアップ手順

1. Docker / Docker Compose をインストールする
2. リポジトリを配置する: `git clone <repo-url> /opt/yomikiri/app`
3. 秘匿情報を配置する
   ```sh
   sudo mkdir -p /etc/yomikiri
   sudo tee /etc/yomikiri/batch.env <<'EOF'
   DATABASE_URL=postgres://...
   EOF
   sudo chmod 600 /etc/yomikiri/batch.env
   ```
4. 状態ディレクトリを用意する: `sudo mkdir -p /var/lib/yomikiri`
5. systemd unit を配置する
   ```sh
   sudo cp /opt/yomikiri/app/deploy/systemd/yomikiri-batch.service \
           /opt/yomikiri/app/deploy/systemd/yomikiri-batch.timer /etc/systemd/system/
   sudo systemctl daemon-reload
   sudo systemctl enable --now yomikiri-batch.timer
   ```
6. Mackerel の check プラグインを配置する（`check-file-age` を含む
   mackerel-check-plugins を別途インストールしておく）。
   `[plugin.checks.*]` は `conf.d/*.conf` の `include` からは読み込まれないため、
   `conf.d/` にはコピーせず `/etc/mackerel-agent/mackerel-agent.conf` の末尾に
   直接追記する。**初回のみ実行すること**（再実行すると内容が重複してしまう）。
   ```sh
   sudo tee -a /etc/mackerel-agent/mackerel-agent.conf < /opt/yomikiri/app/deploy/mackerel/yomikiri-batch-checks.conf
   sudo systemctl restart mackerel-agent
   ```
7. 動作確認: `sudo systemctl start yomikiri-batch.service` で手動起動し、
   `journalctl -u yomikiri-batch -f` でログを確認する

#### 更新手順

```sh
cd /opt/yomikiri/app
./deploy/bin/deploy.sh
```

- 本番 URL でトップページが表示され、投票まで一連の機能が動作する

- main への push で Cloudflare へ自動デプロイされる
