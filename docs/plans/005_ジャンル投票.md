## 実装タスク

### 1. localStorage ユーティリティ

クライアント側の状態を一元管理するモジュールを作る。キー設計の例:

| キー | 内容 |
| --- | --- |
| `anonymousUserId` | `crypto.randomUUID()` で生成した UUID |
| `pendingRead` | `{ oneshotId, clickedAt }`（「読む」クリック時に保存） |
| `votedOneshotIds` | 投票済み作品 ID の配列 |
| `skippedOneshotIds` | スキップ済み作品 ID の配列 |
| `readOneshotIds` | 読了検知済み（既読）作品 ID の配列。カード表示の既読スタイルに使う |
| `favoriteOneshotIds` | お気に入り登録した作品 ID の配列（追加順。/favorites ページで使用） |

- 匿名ユーザー ID は初回アクセス時に生成し、以後使い回す
- JSON パース失敗など壊れた値は破棄して初期化する

### 2. 読了検知

[001_plan.md 7.2 章](../001_plan.md) のフローを実装する。

1. カードのクリック時に `pendingRead` を保存してから新規タブで外部遷移する
2. `visibilitychange`（visible 復帰）とページ再訪問時に `pendingRead` を確認する
3. `clickedAt` から 10 秒以上経過 → 既読として記録し、投票モーダルを表示する
4. 10 秒未満 → 記録を破棄する
5. 投票済み・スキップ済みの作品ではモーダルを表示せず記録を破棄する

- 判定ロジックは純粋関数として切り出し、単体テストを書く
  （経過時間・投票済み・スキップ済みの組み合わせを網羅）

### 3. 投票 API

`POST /api/oneshots/[id]/votes` を Route Handler で実装する。

- リクエストボディを zod でバリデーションする
  - `anonymousUserId`: UUID 形式
  - `genreIds`: 1 件以上の整数配列、実在する `genres.id` であること
- レスポンス
  - `201 Created`: 登録成功（重複は `ON CONFLICT DO NOTHING` で吸収し成功扱い）
  - `400 Bad Request`: ボディ不正
  - `404 Not Found`: 対象の読み切りが存在しない
  - `429 Too Many Requests`: rate limit 超過
- 簡易 rate limit: 同一 IP（`cf-connecting-ip` ヘッダー）からの連続投稿を制限する
  （Workers の環境制約を踏まえ、まずは軽量な実装に留める）

### 4. 投票モーダル UI

[002_design.md 4.5 章](../002_design.md) の仕様で実装する。

- 対象作品のサムネイル（小）+ タイトル、質問文、ジャンルチップ一覧、フッターボタン
- ジャンルチップは複数選択可（プラン 004 のチップコンポーネントを再利用）
- 「投票する」（primary）はジャンル未選択の間 disabled
- 「スキップ」（secondary）で `skippedOneshotIds` に記録して閉じる
- 投票成功で `votedOneshotIds` に記録して閉じる
- 投票失敗時はモーダル内にエラーボックス
  （[002_design.md 4.7 章](../002_design.md)）を表示し、再試行できるようにする
- アクセシビリティ: フォーカストラップ、`Esc` で閉じる（= スキップ扱い）、
  `role="dialog"` + `aria-modal` を付与する

### 5. バッジ表示への反映

- 投票直後はページが ISR キャッシュのため即時反映されないことを許容する
  （仕様通り `revalidate: 300` で最大 5 分遅れ）

## 完了条件

- 「読む」→ 10 秒以上経過して復帰 → 既読記録・モーダル表示、の一連の流れが動作する
- 10 秒未満の復帰・投票済み・スキップ済みの作品ではモーダルが表示されない
- 投票 API が仕様通りのステータスコードを返し、重複投票が DB 上で発生しない
- 読了判定ロジックの単体テストが通る
