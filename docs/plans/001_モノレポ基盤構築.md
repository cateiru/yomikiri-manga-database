# 実装プラン 001: モノレポ基盤構築

## 概要

pnpm workspace による monorepo の土台を構築する。
以降のすべての実装プランはこの構成の上に乗るため、最初に実施する。

- 参照: [001_plan.md 2 章「技術スタック」](../001_plan.md)、[10 章「リポジトリ構成」](../001_plan.md)

## 依存関係

- 依存する先行プラン: なし（最初に実施）
- 後続プラン: 002 以降すべて

## スコープ

| 項目 | 対象 |
| --- | --- |
| pnpm workspace の初期化 | ✅ 対象 |
| apps / packages のディレクトリ雛形 | ✅ 対象 |
| TypeScript / Lint / Format の共通設定 | ✅ 対象 |
| `sources.json` とスキーマの配置 | ✅ 対象 |
| 各パッケージの実装本体 | ❌ 対象外（後続プラン） |

## 実装タスク

### 1. workspace の初期化

- ルート `package.json` を作成する（`private: true`）
- `pnpm-workspace.yaml` を作成し、`apps/*` と `packages/*` を登録する
- `.gitignore` を作成する（`node_modules`、`.env*`、ビルド成果物など）
- `.node-version` などで Node.js のバージョンを固定する（LTS を採用）

### 2. パッケージ雛形の作成

以下の 3 パッケージを空実装（`package.json` + `src/` の雛形）で作成する。

```text
apps/
├── web/      # @yomikiri/web   … Next.js（プラン 004 で実装）
└── batch/    # @yomikiri/batch … バッチ（プラン 003 で実装）
packages/
└── db/       # @yomikiri/db    … Drizzle スキーマ（プラン 002 で実装）
```

- `apps/web` と `apps/batch` は `@yomikiri/db` へ `workspace:*` で依存させる

### 3. 共通ツールチェーン

- TypeScript: ルートに `tsconfig.base.json` を置き、各パッケージが extends する
  - `strict: true` を必須とする
- Lint / Format: ESLint + Prettier（または Biome）をルートで統一管理する
- ルート `package.json` に横断スクリプトを定義する
  - `pnpm lint` / `pnpm typecheck` / `pnpm build`

### 4. sources.json の配置

- リポジトリ直下に `sources.json` を作成する（初期エントリは `comic-days` のみ）
- `sources.schema.json`（JSON Schema）を作成し、`$schema` で参照させる
  - `key` / `name` / `listUrl` / `parser` / `enabled` の型・必須を定義する
    （その後 `listUrl`（単数）は複数の一覧ページを扱えるよう `listUrls`（配列）へ統合され、
    `siteUrl` / `favicon` / 任意項目 `fallbackSourceKey` が追加されている。
    最新のフィールド定義は `sources.schema.json` および
    [001_plan.md 4 章](../001_plan.md) を参照）

## 完了条件

- `pnpm install` が成功する
- `pnpm typecheck` と `pnpm lint` が全パッケージで成功する
- `sources.json` が JSON Schema に対して valid である
- `apps/web`・`apps/batch` から `@yomikiri/db` を import できる（空 export で確認）
