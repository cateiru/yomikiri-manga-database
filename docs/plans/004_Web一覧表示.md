# 実装プラン 004: Web 一覧表示（apps/web）

## 概要

Next.js（App Router）でトップページ（読み切り一覧）を実装する。
デザイントークン・共通コンポーネントもこのプランで整備する。

- 参照: [001_plan.md 6.1 章「トップページ」](../001_plan.md)、[7.3 章「ジャンル集計・表示」](../001_plan.md)、
  [002_design.md](../002_design.md) 全編

## 依存関係

- 依存する先行プラン: [002 DB スキーマ](./002_DBスキーマ.md)
  （表示確認には [003 バッチクローラー](./003_バッチクローラー.md) のデータ投入があるとよい）
- 後続プラン: 005（投票 UI）、006（Cloudflare へのデプロイ）

## 実装タスク

### 1. Next.js セットアップ

- App Router 構成で `apps/web` を初期化する
- `@opennextjs/cloudflare` を導入し、`wrangler.jsonc` を用意する
  （デプロイ自体はプラン 006。ここではローカルビルドが通るところまで）
- DB 接続は `@yomikiri/db/client-serverless` を使用する

### 2. デザイントークン

[002_design.md 3 章](../002_design.md) のトークンをグローバル CSS の
カスタムプロパティとして定義する。

- カラー（`--color-ink` 系 + `--color-error`）
- タイポグラフィ（`--font-sans`、`--text-xs`〜`--text-xl`）
- ボーダー・シャドウ（`--border-width`、`--shadow-sm` / `--shadow-lg`、`--radius: 0`）
- スペーシング（`--space-1`〜`--space-8`）

### 3. 共通コンポーネント

[002_design.md 4 章](../002_design.md) の仕様に従い実装する。

| コンポーネント | 仕様の要点 |
| --- | --- |
| ヘッダー / フッター | 4.6 章。テキストロゴ + 墨色ボーダー |
| 読み切りカード | 4.1 章。hover で浮き上がり、active で沈み込み |
| ジャンルバッジ | 4.2 章。1 位は反転（墨色ベタ塗り）、2〜3 位は標準 |
| ジャンルチップ | 4.4 章。checkbox ベース、選択時に「✓」表示 |
| 空状態表示 | 4.7 章。破線ボーダーの「空のコマ」 |
| サムネイルプレースホルダー | 4.8 章。「NO IMAGE」テキスト |

アクセシビリティ要件（[002_design.md 5 章](../002_design.md)）を必ず適用する。

- フォーカスリングの統一（`outline` 無効化の禁止）
- `prefers-reduced-motion: reduce` 時のアニメーション無効化

### 4. 一覧取得クエリ

- Server Component から Drizzle で取得する
- `genre_votes` を `GROUP BY` 集計し、作品ごとの得票数上位 3 ジャンルを付与する
  - 同数時は `genre_id` 昇順でタイブレーク
- ソート: `COALESCE(published_at, first_seen_at)` の降順
- ページは ISR（`revalidate: 300`）でキャッシュする

### 5. トップページ

- カード grid（レスポンシブ。モバイル 2 列〜デスクトップ 5 列目安）
- grid 上部にジャンルフィルタ（チップの横並び）を配置する
  - 選択状態は URL クエリパラメータ（例: `?genre=battle`）で表現する
- カードクリックで `viewer_url` を新規タブで開く
  （`rel="noopener noreferrer"` を付与。クリック記録の処理はプラン 005 で追加）
- フィルタ結果 0 件時は空状態表示を出す

## 完了条件

- ローカル（`pnpm --filter @yomikiri/web dev`）で一覧が表示される
- ジャンルバッジが得票数上位 3 件、1 位のみ反転スタイルで表示される
- ジャンルフィルタで絞り込みができ、0 件時に空状態が表示される
- サムネイルが無い作品でプレースホルダーが表示される
- `opennextjs-cloudflare build` が成功する
